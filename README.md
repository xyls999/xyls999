### Hehai University Zhize Laboratory

I belong to **Hehai University Zhize Laboratory**, a mainly undergraduate laboratory shaped around technical practice, project exploration, and innovation-oriented learning.

This context gives my work a practical bias. Software should be tested, discussed, documented, and improved through repeated attempts. A project is more convincing when someone else can read it, run it, and understand why it was built that way.

The laboratory atmosphere keeps this profile from becoming a pure resume wall. It should feel like a working bench with selected projects, reproducible notes, learning traces, and technical direction in one place.

## Repository Notes

<p align="center">
  <img width="100%" src="./media/sources/dream-map.jpg" alt="Concept map visual" />
</p>

I prefer repositories that keep the reasoning visible. Code is important, but setup notes, tradeoffs, failed attempts, and reproduction steps are what make a project useful after the first commit.

This is why the profile mixes project links with visual markers and concise explanations. Each section should feel like part of a learning record, not a detached gallery.

My near-term goal is to make selected repositories easier to read, reproduce, and evaluate: clearer README files, smaller verified examples, and more direct notes about what works.
